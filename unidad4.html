Unidad IV: Generación de Código Objeto

Compiladores · Lenguajes · Semántica

Registros

Son espacios de almacenamiento rápidos ubicados dentro del CPU. La asignación de registros implica decidir qué variables se guardan allí para mejorar el rendimiento.

  • Estrategias: Se priorizan las variables más utilizadas.
  • Problema: Cuando no hay suficientes registros, ocurre un spill o "derrame", y los datos deben almacenarse temporalmente en memoria.

Lenguaje Ensamblador

Es una representación legible del código máquina. Cada instrucción tiene una correspondencia directa con operaciones de bajo nivel del procesador.

Ejemplo en x86 para x = a + b:

    
MOV eax, [a]   ; Carga 'a' en el registro eax
ADD eax, [b]   ; Suma 'b' a eax
MOV [x], eax   ; Guarda el resultado en 'x'

Lenguaje Máquina

Es el código binario que el procesador ejecuta directamente. Cada instrucción ensamblador tiene una traducción en lenguaje máquina (ej: 0x89 0xC3 para MOV eax, ebx).

  • Ensamblador: Traduce instrucciones simbólicas a binario.
  • Compilador: Traduce lenguajes de alto nivel a instrucciones en ensamblador.

Administración de Memoria

La memoria se organiza en diferentes áreas con propósitos distintos:

  • Memoria Estática: Para variables globales y constantes.
  • Pila (stack): Usada en llamadas a funciones, parámetros y variables locales.
  • Heap: Para memoria dinámica, gestionada manualmente (ej. malloc en C).

Ejemplo en ensamblador de gestión de pila:

  
PUSH ebp
MOV ebp, esp
SUB esp, 4     ; Reserva espacio para una variable local
...
MOV esp, ebp
POP ebp
RET
